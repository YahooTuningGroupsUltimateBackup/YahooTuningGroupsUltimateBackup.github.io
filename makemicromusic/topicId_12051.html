This quote from the Cameleon5000 Manual explains this as well as I
could:

"One of Cameleon 5000's unique features is its ability to import
standard .wav & .aiff sampled sounds into Voice programs.
It is important to understand that Cameleon 5000 does not load and
play samples in the same way a conventional sample-based synthesiser
would.
In fact it doesn't load sample data at all.
Instead it closely examines the contents of your chosen file (using
Camel Audio's unique Sonic blueprint technology) and analyses the sound.
It then automatically creates a whole new Voice program, with all the
Partials, Envelopes and other parameters set in such a way as to
imitate that sound.
Once a sampled sound has been imported, you can treat it just as you
would any other Voice Program.
It can be morphed with the Morph square, or tweaked and edited just
as if you'd programmed it from scratch yourself."

The tweaking of course can include microtuning using a tuning table.

You can be running up to four of these sounds in one camelaudio plug-
in, and then add more plug-ins to run at the same time.
I have managed to run about four of these plug-ins at the same time
with Logic without any indications of overloading.
You can also have each plug-in running a different tuning table.

Hopefully this will answer your question and stimulate lurkers to
invent even more microtuning possibilities.
