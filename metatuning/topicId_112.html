Incidentally, we needn't be helpless victims here. It's fairly easy
to write a Python script to delete messages from the server if they
exceed a certain size, or come from a certain person. It won't work
with digests, but it's a start.

As an example, here's the script I use to clear any tuning list
messages I've read at work so I don't have to download them again at
home. You can update it to work with whatever mailing lists you're
subscribed to, and whatever spam you regularly get.

import string, re
import getpass, poplib

liststrings = [
"'Mailing-List: list tuning@yahoogroups.com; contact tuning-
owner@yahoogroups.com',",
"'Mailing-List: list practicalmicrotonality@yahoogroups.com;
contact practicalmicrotonality-owner@yahoogroups.com',",
"'Mailing-List: list harmonic_entropy@yahoogroups.com; contact
harmonic_entropy-owner@yahoogroups.com',",
]
spamstrings = ["'From: \"silicon.com\" <silicon@...>',",
"'To: minim@...',"]

box = poplib.POP3('mail.your.isp')
box.user('username')
box.pass_(getpass.getpass())
count, size = box.stat()
for mess in range(1,count-10): # save 10-ish messages
header = str(box.top(mess, 0))
for junkstring in (liststrings+spamstrings):
if string.find(header, junkstring)>=0:
print mess, subject
box.dele(mess)
break
box.quit()
