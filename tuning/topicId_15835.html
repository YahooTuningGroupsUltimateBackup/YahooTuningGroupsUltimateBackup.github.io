I've just uploaded new 1.09 beta.

This can be used to remake the midi clips for the recent VRML models.

To do this, navigate to the folder with the clips, then open the .ts file (prob. called first_tune.ts).

Then change voice, length of note, pitch as desired.

Then type #mmc into the Sustain box (or #mma for arpeggios, #mmb for broken chords).

Then just wait while the clips are made. This will work because the zips for the new VRML models also include the
MIDI_CLIPS file, listing all the midi clips needed, that FTS uses to make them.

For more details, see the beta download page, and:
http://members.nbci.com/tune_smithy/fts_beta/MIDI_CLIPS.htm

Can also be used to make any list of midi clips for chords / broken chords / arpeggios that uses the same file name
conventions. For instance, I plan to use it for sound clips for the Margo Schulter articles. (More options will be added
to the file name convention eventually - see the prev url. for some ideas I have in mind).

This also fixes some bugs in prev. beta, and you can now set the note length to any value you like (prev. was limited to
notes of at most 65 secs approx, = limit of Windows media timer).

Download page
http://members.nbci.com/tune_smithy/fts_beta/fts_beta_download.htm
It's for Windows 95/98

Help needs to be updated with all the new options, before the release proper,

Robert.

Correction.

#mmc etc makes the clips exactly as written, so 3s5 becomes 15/1, etc.

It's #msc (or #msb, #msa) if you want to reduce them all into same octave (or other compass, which can be specified in
file MM_INFO in same directory).
