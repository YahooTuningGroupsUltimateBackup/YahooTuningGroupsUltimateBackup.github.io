I noticed the error u made with the Real Audio and meant to email u about
it. Here's some notes (for anyone interested) re. putting Real Audio on a
web site :

The RealAudio encoder, RealProducer G2 is available for free from
http://www.real.com/ for Windows & Macs. They have a more advanced
shareware version too, but the free RealProducer is more than adequate.
Encode for streaming at 28.8kbps Use the "single stream" option, not "Sure
Stream". And select the "Music" setting, not "stereo music", which is
actually of lower quality. And select "allow to download", if you want
people to be able to download your music to their own computer.

Ideally do what I've done, have one link for streaming :

Example : "http://www.users.interport.net/~dante/alice.ram"

The ".ram" file is actually just a text file with the URL of the
actual Real Audio ".ra" file, for example, the streaming link, in this case
called "alice.ram" is a link to a text file whose only content is one line
"http://www.users.interport.net/~dante/alice.ra", i.e. a link to the actual
music file. The RealPlayer requires this to stream a file correctly.

Then offer another link for those who want to just download the file to
their computer, so that link would just be
"http://members.xoom.com/drew_skyfyre/blink.ra"

Put simply, your link should be to a ".ram" file, which is a plain text file
containing one line of text, i.e. the absolute URL, for example :
"http://www.users.interport.net/~dante/alice.ra"

U have mistakenly named your actual Real Audio files ".ram", they should be
".ra".

Hope that's been as confusing to u as it has been to me !
