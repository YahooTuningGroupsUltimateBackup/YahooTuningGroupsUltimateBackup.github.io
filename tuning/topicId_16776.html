Just to say, I've been able to duplicate this bug, by looking at the
debug logs Joseph has sent to me - thanks!

It happens if you have
"Slide to most recently visited new scale, arp. or seed" ticked
in
Arpeggio / scale playback config button (the two tied quavers to
right of the Scale... button)

If you then use one of the play buttons, with none of the
scale /arp / seed windows showing, FTS tries to slide each note, and
crashes.

The code is intended to be used only when you have two or more seed /
scale / arpeggio windows showing, so it isn't too surprising it would
crash if none are showing, but I will only know the exact details and
be able to fix it when I go home (after Sunday).

To fix it, untick that option.

(if it is disabled, show two of the scale, arp. or seed windows to
enable it).

Alternatively, if you don't mind resetting all the check boxes, just
tick File | Reset Check boxes etc.

It is rather easy to tick a check box without intending to, as all
you need do is to click on its text in order to move the focus back
to the window it is on. This is the way check boxes work in Windows,
so not much I can do about it. But of course, FTS shouldn't crash,
whatever options one ticks!!

What a relief!!
