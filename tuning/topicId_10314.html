Graham is right. If you want middle C to be fixed to 261.63 Hz, you must do
"load/map clinear". You could add this command to the file startup.cmd if
that's convenient. The command "show mapping" will show the current
keyboard
mapping.
Maybe this little shift in frequency contributes to the difference in sound
between 31-tone and 1/4-c. meantone that Joseph noticed.
Another thing to try for assessing the influence of the TX81Z tuning
resolution
is trying a different quantisation before sending the scale to the
synthesizer.
The "send" command rounds to the nearest step. But doing
"quantize/best 768" or
"quantize/consistent 768" before sending may yield a different result.
The latter only makes sense for JI scales.
I tried for 19-31.scl and quantize/best gave a different result than
quantize/round. For meanquar_19 it made no difference.
