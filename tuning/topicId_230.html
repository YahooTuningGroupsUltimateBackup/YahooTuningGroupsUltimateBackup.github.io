Maybe a "Guest List" is a possible alternative for this. (Warning: I
haven't thought this out - the idea may suck.) I have seen Guest Lists that
allow a minimal amount of HTML to be added as part of the guest's message.
So as others view the list, they see pictures, hyperlinks, etc. added by the
guests. Here's an example (some guy added a picture of him with his
motorcycle):

http://saturn.guestworld.tripod.lycos.com/wgb/wgbview.dbm?owner=delphina13

What's good about it is it would run itself (except for occasional "list
management", like deleting out offensive text or something).

>==========
>
>I know nothing, really, about the practicalities of managing a Web page,
>but HTML is a simple scripting, or formatting, language run atop Internet
>protocols, so read/write permissions are an OS (probably Unix) matter.
>Probably someone would need to write an HTML front page, and get permission
>from the host to set r/w access for the users.

I doubt you'll want to give general "write permission" to a page, because a
mistake by any of the users could cause the whole thing to get deleted or
overwritten. For that reason, a page with "append permission" is safer. By
"append", I mean a page that allows users to add stuff but not delete. The
aforementioned Guest Lists work this way.
