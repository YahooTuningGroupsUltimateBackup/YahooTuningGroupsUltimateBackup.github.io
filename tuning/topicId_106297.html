So far I've found two bugs from Yahoo! Groups, both of which I've posted here:

Yahoo Groups CSS specificity bug causes visited links to remain blue:
http://yahoo.uservoice.com/forums/209451-us-groups/suggestions/4410323-yahoo-groups-css-specificity-bug-causes-visited-li

The "..." auto-trim feature doesn't work properly:
http://yahoo.uservoice.com/forums/209451-us-groups/suggestions/4410388-the-auto-trim-feature-doesn-t-work-properly-

Please upvote these with "3 votes" so that Yahoo! will see them and
fix these bugs.

The first one is some free debugging from me explaining to them why
visited links don't change color (a CSS bug).

The second one involves the fact that the auto-trim "feature" doesn't
work, as you can see here:
http://groups.yahoo.com/neo/groups/tuning/conversations/messages/106267

The algorithm has apparently decided that the entire thing is a quote
and needs to be trimmed, yielding every type of post reply hopeless
unless you top-post. A workaround for now is to change the
auto-generated part where it says (I assume auto-trim will kick in
now)
