Yeah, it answers the question. The precise moment when it crashes is when
the selected cell goes into edit mode thus making the cells contents
unavailable for calculation (thus crashing). This will happen when any cell
in the entire workbook goes into edit mode, not just ones containing data.
The easiest way to fix it is to just drop in sliders that control the
generator's contents. That will skip the cell's edit mode and not crash the
app. You'll still have to be careful not to put any cells into edit mode
while the macro is running, but it should allow you to make the changes you
want without stopping and restarting first.
