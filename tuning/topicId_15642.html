I've just found this web site that describes the state of play for VRML on the mac
and makes recommendations of browsers to use etc.
http://w1.2691.telia.com/~u269100246/vr/mac/

And this FAQ
http://w1.2691.telia.com/~u269100246/vr/mac/faq.html
E.g. says you can't use IE for Cosmo Player on the mac, but have to use Netscape.
Also says that CosmoPlayer + Netscape needs rather a lot of free RAM (40 MB min!).
With Worldview as an alternative, but as usual, the WorldView link doesn't work any more.

Also says there that if you have a shape node with no geometry field set
can cause Netscape / Cosmo Player to crash on the Mac.

I have a node in the latest models which has a geometry field with a coord field
and no coordIndex field (which geometry fields often have), so this may also
cause a crash. So I've altered the VRML to remeve it.

Anyone who has been having problems with these models on the Mac may
like to give them another go.

All updated on site and in zip.
