It's okay, the way it is set up, everything will work fine.

Only members of the TL will be able to see the working drafts
in the files area when they use the "Working in progress" tree.

(demo)

However, those files are automatically ftpd to the "On site tree"
whenever it is refreshed.

http://homepage.ntlworld.com/robertwalker/site_ex/tree.htm

Anyone can see them there.

When you link to another faq, either use the absolute url to
the "on site" tree, which you can show using the "hide tree"
link at the top of the page

Or

use the new $$author$tree entry title$bookmark$ system to refer to it.

Don't try a link directly to the file in the files folder as
this will work in the working draft tree, but not in the
on site tree.

That's an important point to make clear.

E.g. to refer to Joseph's new entry on 19-tET notations

use the url to the ON SITE TREE, which at present is
http://homepage.ntlworld.com/robertwalker/site_ex/josephpehrson/19-tET_notations
in the demo tree

or use the
ASCII BOOKMARK

See $$Joseph Pehrson$19-tET notations$$

= $$author$tree entry title$bookmark$

The ascii bookmark ends $$ because you are leaving the bookmark field blank.

When the file is uploaded, my program will convert either into the relative url
../josephpehrson/19-tET_notations
