I was wondering where machine went - looks like the subgroup it's
defined as changed from "2.7.9.11" to "2.9.7.11," putting the 9 before
the 7, presumably because it's a compounded 3 and 3 is less than 7. It
only comes up when you search the 2.9.7.11 subgroup - see here:

http://x31eq.com/cgi-bin/pregular.cgi?limit=2.9.7.11&error=5.0

But not if you search for 2.7.9.11:

http://x31eq.com/cgi-bin/pregular.cgi?limit=2.7.9.11&error=5.0

The same thing happens if you search 2.5.3 instead of 2.3.5:

http://x31eq.com/cgi-bin/pregular.cgi?limit=2.5.3&error=5.0

I suggest it might be useful, when a search is conducted, to
internally place the incoming limit into some sort of normal form, so
that 2.7.9.11 and 2.9.7.11 become equivalent. Placing the numbers in
ascending order would probably do the trick. Otherwise it's going to
get hairy when searching subgroups.

-Mike
