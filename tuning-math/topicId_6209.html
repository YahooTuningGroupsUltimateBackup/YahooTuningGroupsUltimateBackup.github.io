Certainly it is. Each note is 11 times what it would be if I had asked
Scala to render a normal 12-et file. In general, if you take a Scala
file in n-et, multiply each note by an integer m, and then change the
header so that you are in n*m et, the result is exactly the same. If
you change the header so that you are in n*m+1 or n*m-1 et, you get
octave-adjusted renditions. Manuel could probably easily add an octave
adjuster to Scala retuning, actually.
